1. В чем разница между Forward Proxy (обычный прокси) и Reverse Proxy (обратный прокси)?
    Forward Proxy защищает клиента, скрывая его от интернета, а Reverse Proxy защищает сервер, принимая все запросы на себя и распределяя их внутри сети.
2. Что такое Upstream в терминологии Nginx?
    Upstream в Nginx — это именованная группа бэкенд-серверов, на которые прокси-сервер перенаправляет входящий трафик для обработки.
3. Зачем скрывать реальные IP-адреса сервисов от клиента?
    Скрывать реальные IP нужно для безопасности (защита от прямых DDoS-атак) и гибкости, чтобы можно было менять серверы без изменения адреса для пользователя.
4. Какие еще задачи, кроме маршрутизации, может выполнять API Gateway (назовите хотя бы 2)?
    Кроме маршрутизации, API Gateway часто берет на себя авторизацию пользователей и ограничение частоты запросов (Rate Limiting) для защиты от перегрузок.
5. Что произойдет, если Gateway упадет? Как можно от этого защититься?
    Если единственный Gateway упадет, вся система станет недоступна, поэтому для защиты используют кластеры из нескольких шлюзов и технологию виртуального IP для автоматического переключения на живой узел.
